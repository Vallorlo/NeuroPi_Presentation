BCI Backend: Database Models & Data Management

🗄️ Sophisticated Data Model Architecture

Advanced database models supporting complex BCI workflows while maintaining data integrity and enabling efficient queries across different experimental paradigms with comprehensive metadata storage and lifecycle management.

SessionData Model

Foundation for EEG data management with comprehensive metadata storage and file handling capabilities.

class SessionData(models.Model): id = models.UUIDField(primary_key=True, default=uuid.uuid4) user = models.ForeignKey(User, on_delete=models.CASCADE) name = models.CharField(max_length=255) approach = models.CharField(max_length=50, choices=APPROACH_CHOICES) session_file = models.FileField(upload_to='sessions/') sampling_rate = models.FloatField(default=128.0) channels = models.JSONField(default=list) classes = models.JSONField(default=list) total_samples = models.IntegerField(default=0)
User association with cascade deletion
Approach-specific categorization (Motor Imagery/P300)
Automatic metadata extraction via Django signals
JSON fields for flexible channel and class configuration
Data quality metrics and validation

TrainedModel Management

Comprehensive model lifecycle management supporting multiple ML approaches with version control and performance tracking.

class TrainedModel(models.Model): id = models.UUIDField(primary_key=True, default=uuid.uuid4) user = models.ForeignKey(User, on_delete=models.CASCADE) approach = models.CharField(max_length=50, choices=APPROACH_CHOICES) model_file = models.FileField(upload_to='models/') config = models.JSONField(default=dict) performance_metrics = models.JSONField(default=dict) is_active = models.BooleanField(default=False) created_at = models.DateTimeField(auto_now_add=True)
Atomic model activation with race condition prevention
Approach-specific configuration storage (JSON)
Performance tracking and validation metrics
Version control with creation timestamps
Automatic deactivation of existing models

PredictionSession Infrastructure

Real-time BCI applications through comprehensive session management and state tracking with automatic timeout handling.

class PredictionSession(models.Model): id = models.UUIDField(primary_key=True, default=uuid.uuid4) user = models.ForeignKey(User, on_delete=models.CASCADE) model = models.ForeignKey(TrainedModel, on_delete=models.CASCADE) name = models.CharField(max_length=255) prediction_interval = models.FloatField(default=8.0) window_duration = models.FloatField(default=2.0) status = models.CharField(max_length=20, choices=STATUS_CHOICES) started_at = models.DateTimeField(null=True, blank=True) stopped_at = models.DateTimeField(null=True, blank=True)
Sophisticated session lifecycle management
Configurable prediction intervals and window durations
Status tracking with temporal information
Automatic timeout handling and resource cleanup
Performance metrics and usage pattern analysis
